# 我做了一个"输入行业名，AI 30 秒出深度报告"的工具，解析成功率 98%

> 一个面向 AI 产品经理的行业认知加速器，覆盖 40+ 细分赛道，零 API Key 开箱即用。

---

## 一、AI 产品经理的"行业认知"痛点

做过 AI 产品的人都知道一个魔咒：**你永远不可能只做一个行业。**

上周你还在调研 AI + 医疗的影像辅助诊断，这周老板让你评估"覆铜板"的 AI 机会，下周客户问"精密减速器"赛道怎么样。

怎么办？

- **咨询报告**：贵（动辄几万）、慢（等两周出 PDF）、不切题（不知道 PM 到底关心什么）
- **搜索引擎**：碎片化信息，没有判断力，更没有结构化分析
- **问同事/问朋友**：得碰运气，而且行业壁垒越高越难问到靠谱的人

我做了 **KnowHow**——输入任意行业名，AI 自动生成 L1（概览）→ L2（产业链）→ L3（洞察）+ 知识测验的一份完整报告。**30 秒。**

项目地址：👉 [GitHub - xiaolong-shuang/knowhow](https://github.com/xiaolong-shuang/knowhow)

## 二、它能做什么？

### 🤖 AI 行业分析：输入什么，出什么

在搜索框打字（如"覆铜板""固态电池""智能电表""AGV"），AI 生成四板块：

| 层次 | 内容 | 示例 |
|------|------|------|
| **L1 · 行业概览** | 5 核心指标（市场规模/CAGR/集中度/壁垒/市占率）+ 波特五力 + AI 机会矩阵 | "2024年全球市场规模 386 亿美元，CAGR 6.2%，AI 渗透率 <5%" |
| **L2 · 产业链结构** | 上中下游可视化价值地图（价值占比 + 毛利率 + AI 渗透率标注）+ 四类玩家格局 | "上游电子元器件毛利率 8-22%，中游制造 15-35%" |
| **L3 · 深度洞察** | 结构性趋势 + 圈外人认知误区 + 可操作建议 | "AI 渗透率不足 5%，产线质检和预测性维护是新蓝海" |
| **知识测验** | 3 道选择题，答案在前文中可验证 | 帮你自测：是真的理解了这个行业还是只是看了一遍 |

### ⚖️ AI 行业对比：赛道选型利器

输入两个行业（如"新能源 VS 半导体"）：

- **5 维度判决**：市场规模 / 增速 / AI 就绪度 / 壁垒强度 / 综合建议，每个维度的 winner 带理由
- **决策矩阵**：不同优先级（确定性/天花板/速赢/长期）下分别推荐哪个赛道
- **AI 机会矩阵**：每个行业的 Top AI 机会点含成熟度评分

### 💬 智能问答：全页面上下文感知

侧滑聊天面板根据**当前页面**自动切换推荐问题——行业详情页推该行业的问题、分析页推正在搜索的行业、首页推通用高频问题。支持 Markdown 渲染（表格/列表/粗体），流式对话体验。

## 三、技术架构

```
Next.js 16 (App Router + Turbopack)
├── /api/analyze    ← AI 行业分析 API（JSON 结构化输出）
├── /api/compare    ← AI 行业对比 API
├── /api/chat       ← AI 聊天 API（流式对话）
├── LLM Provider 层 ← Anthropic / OpenAI / DeepSeek / 自定义兼容
└── 静态 SSG 页面   ← 3 个精选行业独立页面
```

**多 LLM 支持**：Anthropic Claude · OpenAI GPT-4o · DeepSeek · 阿里百炼 · 硅基流动 · 火山引擎 · Ollama

**零依赖降级**：不配任何 API Key 也能跑——AI 自动切换到本地模拟模式，所有功能工作正常。

## 四、7 层 JSON 解析兜底：从不稳定的 LLM 输出中稳拿结构化数据

这是我做得最花心思的部分。

LLM 输出 JSON 时常因 token 截断而损坏——尤其是行业分析这种长文本场景（5 个指标 + 产业链 + 玩家 + 趋势 + 误区 + 测验 + 来源，随便破 8000 tokens）。遇到截断，一个缺 `}` 的 JSON 直接导致整个分析失败。

我设计了一套 **渐进式 7 层修复管线**：

| Layer | 机制 | 成本 |
|-------|------|------|
| **L0** | System Prompt 内建 token 预算 ≤ 12000，每个描述字段 ≤ 30 字 —— 源头遏制 | 0 |
| **L1-L4** | 标准解析 — 直接 parse → fence 提取 → brace 定位 → 轻量修复（去尾逗号/引号替换） | 0 |
| **L5** | 扫描未闭合 `{}[]"`，自动补齐尾部括号 —— 修复 token 截断 | 0 |
| **L6** | 正则逐字段提取（metrics/players/opportunities/trends/myths/quiz/sources）—— JSON 全毁也能救回核心数据 | 0 |
| **L7** | 缩短版 Prompt 重试（~60% 长度，无 quiz/compareData，字段 ≤ 20 字） | ~30s/次 |

这套防御体系的核心思路是：**前 6 层零额外成本，只在最后一层才付出重试的延迟代价。**

### 实测数据

我准备了 86 条行业样本为测评集，覆盖 40+ 细分领域：

```
解析成功率:  84/86 (98%)
平均延迟:    22.1s
指标完整性:  5.0/5
五力完整:    100%
模糊词(较大/较快):    0/86 条
安全措辞(取决于/需要观察): 0/86 条
```

失败案例只有 2 个——"冷链物流"和"合成生物学"——都是 DeepSeek 模型层面输出了一些非常规结构的文本。**98% 的稳定性在 LLM 结构化输出场景中，我觉得够了。**

测评脚本和 90 条用例全部开源在仓库的 `eval/` 目录下，可以直接 `node eval/run.mjs` 跑。

## 六、怎么玩

```bash
git clone https://github.com/xiaolong-shuang/knowhow.git
cd industry-knowhow
pnpm install
pnpm dev  # → http://localhost:3101
```

不用配任何 Key，打开就能用。

**配 DeepSeek Key 后获得完整体验**（性价比最高，一次分析花几分钱）：

```bash
cp .env.example .env.local
# 编辑 .env.local 填写:
# LLM_PROVIDER=deepseek
# LLM_API_KEY=sk-你的key
```
